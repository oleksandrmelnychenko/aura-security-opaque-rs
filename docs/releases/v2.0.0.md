# Aura OPAQUE v2.0.0 — Rebrand Release

**Date:** 2026-04-16
**Status:** Breaking change release

## Summary

Full rename from the previous `ecliptix-opaque-rs` project (Swift module `EcliptixOPAQUE`) to **`aura-security-opaque-rs`** (Swift module `AuraOPAQUE`), backing the `auramessenger.ai` product. Every identifier, module name, and artifact name has changed. The underlying Rust cryptographic implementation (hybrid PQ-OPAQUE: 4DH Ristretto255 + ML-KEM-768) is unchanged.

## Breaking changes

### Package / module

| Layer | Before | After |
|---|---|---|
| GitHub repo | `ecliptix-opaque-rs` | `aura-security-opaque-rs` |
| Swift Package | `EcliptixOPAQUE` | `AuraOPAQUE` |
| Swift source dir | `swift/Sources/EcliptixOPAQUE/` | `swift/Sources/AuraOPAQUE/` |
| Binary target | `EcliptixOPAQUEBinary` | `AuraOPAQUEBinary` |
| Swift target | `EcliptixOPAQUESwift` | `AuraOPAQUESwift` |
| xcframework asset | `EcliptixOPAQUE.xcframework.zip` | `AuraOPAQUE.xcframework.zip` |

### Retained

Rust workspace crate names are unchanged (no brand prefix): `opaque-core`, `opaque-agent`, `opaque-relay`, `opaque-ffi`. The FFI static library `libopaque_ffi.a` and all C header filenames/symbols in `rust/include/` also remain unchanged.

### FFI surface

The public C ABI is **unchanged** in v2.0.0. Consumers integrating via `libopaque_ffi.a` or `rust/include/*.h` do not need to modify their code when updating from v1.0.1 — only Swift Package consumers must update their module import from `import EcliptixOPAQUE` to `import AuraOPAQUE`.

## Consumer Notes

- Swift consumers must update the module import from `EcliptixOPAQUE` to `AuraOPAQUE`.
- C and Rust consumers do not need code changes — the ABI and crate names are preserved.
- Relay-side consumers must continue passing the exact same `account_id` used on the agent side.
- Use `opaque_error_string()` for a stable human-readable description of return codes.

## Verification

After rebuild:

```bash
cargo build --release --manifest-path rust/Cargo.toml
cargo test --manifest-path rust/Cargo.toml
cargo clippy --manifest-path rust/Cargo.toml --all-targets -- -D warnings
```

Then rebuild the xcframework, publish as a GitHub release asset, and verify `Package.swift` `checksum:` matches the SHA-256 of the uploaded zip.
