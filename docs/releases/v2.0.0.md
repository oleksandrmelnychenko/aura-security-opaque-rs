# Ecliptix OPAQUE v2.0.0

This release publishes the cleaned major-version Apple binary and C header surface for the current OPAQUE implementation.

## Highlights

- Canonical public headers now match the shipping Rust FFI signatures.
- Agent-side FFI uses direct return-code error handling instead of the old structured `out_error` layer.
- `opaque_agent_generate_ke1` requires explicit `account_id` binding.
- Swift imports the packaged binary module directly and exposes `generateKE1(password:accountId:state:)`.
- Release packaging continues to ship curated headers, `module.modulemap`, XCFramework checksum, and build provenance.

.binaryTarget(

## Consumer Notes

- C and Swift consumers must update any agent-side calls that previously relied on implicit account context or structured `out_error` parameters.
- Relay-side consumers must continue passing the exact same `account_id` used on the agent side.
- Use `opaque_error_string()` for a stable human-readable description of return codes.
